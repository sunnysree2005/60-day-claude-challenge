# Day 37: Task Compass 🧭

Welcome to my submission for **Day 37** of the Interactive Prompt Engineering challenge! Today's task focused on building **"Task Compass"**—a management simulation web application built with Claude that teaches ownership, workflow routing, cross-functional collaboration, and organizational thinking through gamified scenarios.

---

## 🚀 Project Overview
Task Compass moves away from static quizzes to simulate how work actually flows through real organizations. By taking on realistic workplace challenges, players build an intuitive grasp of corporate mechanics and team dynamics.

### Key Features:
* **Interactive Workplace Selection:** Tailors the simulation context (Tech Company, Startup, Corporate, etc.).
* **Three-Stage Simulation:**
  * *Stage 1 (Who Owns This?):* Assigning primary responsibility and understanding supporting roles for everyday workplace tasks.
  * *Stage 2 (Task Routing):* Building structured workflows by sequencing organizational handoffs.
  * *Stage 3 (Collaboration Challenge):* Tackling complex, multi-departmental problems requiring broad coordination.
* **Organizational Thinking Dashboard:** Replaces arbitrary scores with a deep behavioral breakdown measuring Ownership, Delegation, Collaboration, and Workflow Thinking using a custom radar/bar visualization.
* **Final Reflection Journal:** Generates tailored insights regarding management tendencies and organizational behavior.

---

## 🛠️ Step-by-Step Implementation

1. **Prompt Execution:** Fed the Task Compass prompt template into Claude to generate a robust, self-contained, single-file modern dark-mode HTML application (`index.html`).
2. **Local Testing:** Launched the application locally to test drag-and-drop role assignments, stage transitions, animated workflow routing, and final analytical rendering.
3. **Documentation:** Captured gameplay screenshots for the repository portfolio.
