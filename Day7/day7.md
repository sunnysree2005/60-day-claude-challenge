## 📊 My Claude Usage Strategy

### 🚀 Recommended Primary Model: Claude 3.5 Sonnet
* **Why it fits:** As an ETE student heavily engaged in programming (C++/Python), DSA implementations, and hardware simulation logic, Claude 3.5 Sonnet provides the ideal matrix of top-tier coding intuition, speed, and precision.

### ⚙️ Model Allocation Guide
* **Haiku:** Best utilized for rapid syntax checking, quick logic reviews, or defining simple engineering terminologies.
* **Sonnet:** The daily driver for architecting complex algorithms, writing structural code blocks, and calculating filter or Op-Amp parameters.
* **Opus:** Strictly reserved for deep academic research papers, cross-referencing expansive technical documentations, or complex ML mathematical modeling.

### 💡 Recommended Effort Levels Table

| Task | Best Model | Best Effort | Reason |
| :--- | :--- | :--- | :--- |
| **Quick Bug Fixing & Code Compilation Syntax** | Claude Haiku | Low / Standard | Prevents burning high-tier model message counts on trivial tasks. |
| **DSA Algorithm Implementations & Code Snippets** | Claude Sonnet | Standard | Generates clean, well-commented, and efficient data structures quickly. |
| **Complex Math Formulation & Machine Learning Logic** | Claude Sonnet | High | Allocates deep token chains to systematically step through complex logical weights. |
| **Deep Project Research & Capstone Hardware Planning** | Claude Opus | Max | Grants maximum reasoning capabilities for highly ambiguous or intricate concepts. |

---

## ❌ Biggest Mistakes to Avoid
1. **Using Maximum Effort on Trivial Scripts:** Squandering your message count or time limits by turning on High/Max reasoning for basic code bugs that Haiku can instantly fix.
2. **Homogeneous Interaction Profiles:** Failing to manually downscale your environment choices when switching tasks from deep engineering down to simple textual adjustments.

---

## 💡 Key Learnings
1. **Reasoning Budget Management:** AI attention is a precious, bounded resource. Keeping a live eye on telemetry via the "Claude Counter" interface is highly essential to prevent mid-task lockout during intense study periods.
2. **Context-Driven Processing:** High or Max reasoning profiles shouldn't be left running uniformly; they must be weaponized purposefully when handling abstract hardware mechanics or heavy mathematics to achieve optimal execution.
